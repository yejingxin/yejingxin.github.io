FROM -> WHERE -> AS Excuting Order Why the defined colmn name cannot be used in WHERE clause? Because the col AS new_col is excuted after WHERE. The excuting order in the follow SQL is FROM -> WHERE -> AS: SELECT col AS new_col FROM table WHERE col < 10 EXISTS operator The EXISTS operator is used to test for the existence of any record in a subquery, it returns TRUE if the subquery returns one or more rows.
